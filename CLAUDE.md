# Dotfiles

Max's macOS dotfiles, managed with GNU Stow.

## How Stow Works

Each top-level directory is a **stow package** that mirrors the target file structure relative to `~`. Stow creates symlinks from `~` into this repo. Since everything is symlinked, editing files in this repo directly changes the active config.

The `--dotfiles` flag in `.stowrc` means files named `dot-*` become `.*` when symlinked. For example:

- `zsh/dot-zshrc` тЖТ `~/.zshrc`
- `git/dot-gitconfig` тЖТ `~/.gitconfig`
- `p10k/dot-p10k.zsh` тЖТ `~/.p10k.zsh`
- `brewfile/dot-config/brewfile/Brewfile` тЖТ `~/.config/brewfile/Brewfile`
- `worktrunk/scripts/worktrunk/wt-zellij-tab` тЖТ `~/scripts/worktrunk/wt-zellij-tab`

To add a new config:

1. Create a new directory at the repo root (e.g. `aerospace/`)
2. Inside it, mirror the path structure relative to `~` using `dot-` prefix for dotfiles
3. Add `stow <dirname>` to `setup.sh`
4. Run `stow <dirname>` to create the symlink

Re-stow is only needed after adding new files/packages, not after edits.

## Tools

When editing configuration for a tool, read its docs before making changes. Don't guess at option names or values.

  - Man page: `/opt/homebrew/share/man/man8/stow.8`
  - Detailed topics (tree folding, conflicts, ignore lists): `/opt/homebrew/share/doc/stow/manual-split/`

- **Homebrew** тАФ Package manager.
  - Config: `brewfile/dot-config/brewfile/Brewfile`
  - Man page: `/opt/homebrew/share/man/man1/brew.1`
  - Extended docs: `/opt/homebrew/share/doc/homebrew/`

- **homebrew-file** (`brew-file`) тАФ Used for `brew file install` on a fresh machine. The Brewfile is manually curated.
  - No local docs тАФ run `brew file help` for usage.

- **Defuddle** тАФ Extract article content and metadata from web pages.
  - No man page or bundled docs тАФ use `defuddle --help` for usage.

- **Antidote** тАФ Zsh plugin manager.
  - Config: `zsh/dot-zsh_plugins.txt`, loaded via `antidote load` in `.zshrc`
  - Man page: `/opt/homebrew/share/man/man1/antidote.1`

- **Powerlevel10k** тАФ Zsh prompt theme.
  - Config: `p10k/dot-p10k.zsh` (generated by `p10k configure`). Heavily commented тАФ serves as its own documentation.

- **Ghostty** тАФ Terminal emulator.
  - Config reference: `/Applications/Ghostty.app/Contents/Resources/ghostty/doc/ghostty.5.md`
  - CLI actions: `/Applications/Ghostty.app/Contents/Resources/ghostty/doc/ghostty.1.md`
  - Check a specific option's default: `ghostty +show-config --default --docs`

- **glow** тАФ Terminal markdown renderer (charmbracelet).
  - No config file; uses defaults
  - No man page тАФ use `glow --help`
  - Online docs: https://github.com/charmbracelet/glow
  - Bound in Helix as `space M` to open the current file in a new zellij pane (`glow -p`)

- **navi** тАФ Interactive cheatsheet tool for the terminal.
  - Config: `navi/dot-config/navi/config.yaml`
  - Cheats: `navi/dot-local/share/navi/cheats/` (one `.cheat` file per tool)
  - No man page тАФ use `navi --help`
  - Bound to `Alt+?` in Zellij as a floating pane

- **fnm** тАФ Fast Node.js version manager.
  - Config: `zsh/dot-zshrc` initializes `fnm`; `zsh/dot-zshenv` exposes the default Node toolchain to non-interactive shells
  - No local docs тАФ use `fnm --help` and online docs at https://github.com/Schniz/fnm

- **Worktrunk** тАФ Git worktree manager for parallel AI agent workflows.
  - Config: `worktrunk/dot-config/worktrunk/config.toml`
  - Scripts: `worktrunk/scripts/worktrunk/` тЖТ `~/scripts/worktrunk/`
  - No man page тАФ use `wt --help` and subcommand help (e.g. `wt switch --help`)
  - Online docs: https://worktrunk.dev/
  - Completions: `/opt/homebrew/share/zsh/site-functions/_wt`

- **yazi** тАФ TUI file manager.
  - Config: `yazi/dot-config/yazi/keymap.toml`
  - Scripts: `yazi/scripts/yazi/` тЖТ `~/scripts/yazi/`
  - No man page тАФ use `yazi --help`
  - Online docs: https://yazi-rs.github.io/
  - Bound in Helix as `space e` тАФ opens yazi as an overlay in the current Helix instance, and the file selected in yazi is opened in that same Helix instance via yazi's `--chooser-file` flag. Requires Helix тЙе 25.07 for the `%{buffer_name}` and `%sh{...}` expansions.

- **Zellij** тАФ Terminal multiplexer.
  - Config: `zellij/dot-config/zellij/config.kdl`
  - No man page тАФ use `zellij --help` and `zellij setup --dump-config` for the full default config
  - Completions: `/opt/homebrew/share/zsh/site-functions/_zellij`

## Adding a New Tool

Checklist when adding a new CLI tool or app to this setup:

1. **Install** тАФ `brew install <name>` (or `brew install --cask <name>` for GUI apps)
2. **Brewfile** тАФ Add the entry to `brewfile/.config/brewfile/Brewfile` in alphabetical order under the correct tap/section
3. **Stow package** тАФ If the tool has config files, create a stow package:
   - Create `<name>/` at the repo root, mirroring the path structure relative to `~`
   - Use `dot-` prefix for dotfiles (e.g. `dot-config` becomes `.config`)
   - Add `stow <name>` to `setup.sh` (alphabetical within the stow block)
   - Run `stow <name>` to create the symlinks
4. **Local docs** тАФ Find local documentation and add it to the Tools section below:
   - Check for man pages: `man -w <name>` (gives the path if one exists)
   - Check for bundled docs: look in `/opt/homebrew/share/doc/<name>/`, the app bundle, or run `<name> --help`
   - Check for completions: `ls /opt/homebrew/share/zsh/site-functions/_<name>`
5. **Tools entry** тАФ Add an entry to the **Tools** section in this file with: a short description, config path (if stow-managed), and local doc paths found above
6. **Things to Know** тАФ If the tool has ordering constraints, gotchas, or interactions with other tools, add a bullet to the **Things to Know** section

Skip steps that don't apply (e.g. no stow package needed for a tool with no config).

## Things to Know

- Zellij autostart **must** stay at the very top of `.zshrc` (before p10k instant prompt) тАФ it launches zellij and exits the outer shell, so nothing after it runs in that shell.
- The `.zshrc` sources `~/.localrc` at the end for machine-specific config that shouldn't be committed.
- `zoxide` init **must** stay last in `.zshrc` тАФ it overrides `cd` and the `chpwd` hook.
- P10k instant prompt **must** stay at the top of `.zshrc`.
- Worktrunk shell integration **must** come before zoxide in `.zshrc` тАФ it needs the shell wrapper for `wt switch` to change directories, but doesn't conflict with zoxide's `cd` override.
- Non-interactive Node access depends on `fnm default` тАФ `.zshenv` prepends `~/.local/share/fnm/aliases/default/bin`, so changing the global default should be done with `fnm default <version>`.
- Plugin order in `dot-zsh_plugins.txt` matters: `fzf-tab` before `autosuggestions`.
- **Stow uses tree folding** тАФ directories like `~/.config/zellij` may be symlinks to the stow source, not real directories. Running `rm` on files inside them deletes the stow source files. Never `rm` files in stow-managed paths to "clean up before restowing." If you need to restow, use `stow -R <package>` instead.
