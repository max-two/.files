# Dotfiles

Max's macOS dotfiles, managed with [GNU Stow](https://www.gnu.org/software/stow/manual/stow.html).

## How It Works

### GNU Stow

Each top-level directory is a **stow package** that mirrors the target file structure relative to `~`. Stow creates symlinks from `~` into this repo.

The `--dotfiles` flag in `.stowrc` means files named `dot-*` become `.*` when symlinked. For example:

- `zsh/dot-zshrc` → `~/.zshrc`
- `git/dot-gitconfig` → `~/.gitconfig`
- `p10k/dot-p10k.zsh` → `~/.p10k.zsh`
- `brewfile/.config/brewfile/Brewfile` → `~/.config/brewfile/Brewfile`

### Adding a new config

1. Create a new directory at the repo root (e.g. `aerospace/`)
2. Inside it, mirror the path structure relative to `~` using `dot-` prefix for dotfiles
3. Add `stow <dirname>` to `setup.sh`
4. Run `stow <dirname>` to create the symlink

### Editing configs

Since everything is symlinked, editing files in this repo directly changes the active config. No re-stow needed after edits — just after adding new files/packages.

## Directory Layout

```
.files/
├── brewfile/           # ~/.config/brewfile/Brewfile (Homebrew packages)
├── ghostty/            # ~/.config/ghostty/config.ghostty
├── git/                # ~/.gitconfig
├── p10k/               # ~/.p10k.zsh (Powerlevel10k prompt theme)
├── zsh/                # ~/.zshrc and ~/.zsh_plugins.txt
├── .stowrc             # Stow config: --dotfiles --target=~
└── setup.sh            # Full machine setup script
```

## Key Tools

- **Homebrew** — package manager. Packages tracked in `brewfile/.config/brewfile/Brewfile`.
- **homebrew-file** (`brew-file`) — automatically keeps the Brewfile in sync when you `brew install`/`brew uninstall`. The `.zshrc` sources `brew-wrap` to enable this.
- **Antidote** — zsh plugin manager. Plugins listed in `zsh/dot-zsh_plugins.txt`, loaded via `antidote load` in `.zshrc`.
- **Powerlevel10k** — zsh prompt theme. Config in `p10k/dot-p10k.zsh` (generated by `p10k configure`).

## Things to Know

- The `.zshrc` sources `~/.localrc` at the end for machine-specific config that shouldn't be committed.
- `zoxide` init **must** stay last in `.zshrc` — it overrides `cd` and the `chpwd` hook.
- P10k instant prompt **must** stay at the top of `.zshrc`.
- Plugin order in `dot-zsh_plugins.txt` matters: `fzf-tab` before `autosuggestions`.
- The `HOMEBREW_BREWFILE_LEAVES=1` export means the Brewfile only tracks top-level packages, not dependencies.
- The git config uses `master` as the default branch name.

## Tool Documentation

When editing configuration for any of these tools, read the relevant docs before making changes. Don't guess at option names or values.

- **Ghostty** — Terminal emulator. Config reference and all options: `/Applications/Ghostty.app/Contents/Resources/ghostty/doc/ghostty.5.md`. CLI actions and flags: `/Applications/Ghostty.app/Contents/Resources/ghostty/doc/ghostty.1.md`. To check a specific option's current default: `ghostty +show-config --default --docs`.
